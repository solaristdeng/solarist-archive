# 文章类型样式功能使用说明

## ✨ 功能介绍

现在你可以为不同类型的文章设置不同的 CSS 样式了！只需在 Markdown 文件的 YAML 头部添加 `type` 字段即可。

## 📝 使用方法

### 基础用法

在你的 `.md` 文件顶部的 YAML 区域添加字段：

**方式一：简单格式（推荐）**
```yaml
---
title: 我的文章标题
tags: 标签1, 标签2, 标签3
type: technical
date: 2025-11-19
---
```

**方式二：YAML 数组格式**
```yaml
---
title: 我的文章标题
tags:
  - "标签1"
  - "标签2"
  - "标签3"
type: technical
date: 2025-11-19
---
```

两种格式都支持，效果相同。简单格式更简洁，数组格式在标签很多或包含特殊字符时更清晰。

**支持的字段**：
- `title` - 文章标题（必需）
- `tags` - 标签（可选，支持上述两种格式）
- `type` - 样式类型（可选，默认为 default）
- `date` - 发布日期（可选，会自动从文件创建时间获取）

注：如果你的 frontmatter 使用 `created` 或 `published` 字段，也会被识别为日期。

### 可用的样式类型

#### 1. **technical** - 技术文档
- 黑底绿字，终端风格
- 适合：技术博客、代码文档、教程
- 示例：`test.md`

```yaml
type: technical
```

#### 2. **literary** - 文学作品
- 优雅衬线字体（Georgia）
- 大行距，首行缩进
- 适合：散文、诗歌、小说
- 示例：`example-literary.md`

```yaml
type: literary
```

#### 3. **legal** - 法律文档
- Times New Roman 字体
- 正式排版，居中标题
- 适合：法律条文、正式文件
- 示例：`中华人民共和国仲裁法_中国人大网.md`

```yaml
type: legal
```

#### 4. **poetry** - 诗歌样式
- 楷体字体，优雅排版
- 极小段落间距（模拟诗行）
- 两次换行形成诗节分隔
- 适合：现代诗、古诗词、诗歌创作
- 示例：`example-poetry.md`

```yaml
type: poetry
```

#### 5. **default** - 默认样式
- 如果不指定 type，或者指定为 default
- 使用原有的 brutalist 风格

```yaml
type: default
# 或者不写 type 字段
```

## 🎨 自定义新样式

想要创建自己的样式类型？很简单：

### 步骤 1：创建 CSS 文件

在 `src/` 目录下创建新文件，命名为 `post-{你的类型}.css`

例如创建新闻样式：`src/post-news.css`

```css
/* 新闻样式 */
.markdown-body {
  font-family: Arial, sans-serif;
  color: #333;
}

.markdown-body h1 {
  color: #c00;
  font-size: 2.5rem;
  border-bottom: 3px solid #c00;
}

.markdown-body p {
  font-size: 1.1rem;
  line-height: 1.8;
}

/* ... 更多样式 ... */
```

### 步骤 2：在文章中使用

```yaml
---
title: 今日头条
type: news
---
```

就这么简单！

## 💡 样式设计建议

设计自己的文章样式时，可以修改：

- **字体**：`font-family`
- **颜色**：`color`, `background-color`
- **间距**：`line-height`, `margin`, `padding`
- **标题样式**：`h1`, `h2`, `h3` 的外观
- **链接样式**：`a` 标签的效果
- **代码块**：`code`, `pre` 的样式
- **引用**：`blockquote` 的外观

记得使用 `.markdown-body` 作为父选择器，避免影响网站其他部分。

## 🛠️ 技术说明

- CSS 文件在打开文章时动态加载
- 切换文章时自动卸载旧样式，加载新样式
- 如果指定的样式文件不存在，会回退到默认样式
- 所有样式都会继承基础的 `article.css`

## ❓ 常见问题

**Q: type 字段区分大小写吗？**
A: 不区分。`Technical`、`technical`、`TECHNICAL` 都可以。

**Q: 可以同时使用多个样式吗？**
A: 目前一次只能指定一个 type。如需组合效果，请创建新的 CSS 文件。

**Q: 样式不生效怎么办？**
A: 检查 CSS 文件名是否正确（`post-{type}.css`），确保文件在 `src/` 目录下。

**Q: 能否修改现有样式？**
A: 当然！直接编辑对应的 CSS 文件即可。

## 📅 文章排序与日期（自动化）

**自动化日期管理**：
- 系统会**自动读取文件的创建时间和修改时间**
- 你不需要在 markdown 中手动输入日期
- 文件创建时间作为发布时间
- 文件最后修改时间会自动更新

**排序规则**：
- 首页文章按**发布时间从新到旧**自动排序
- 排序精确到**分钟**级别（即使一天内发布多篇也能正确排序）
- 没有时间信息的文章排在最后

**日期显示**：
- 首页：悬停在文章上时，显示发布日期（YYYY-MM-DD 格式）
- 文章页左侧信息栏：
  - PUBLISHED: 发布日期
  - MODIFIED: 最后修改日期

**手动指定日期（可选）**：
如果你想手动指定日期，可以在 frontmatter 中添加：

```yaml
---
title: 我的文章
date: 2025-11-19T10:30:00
modified: 2025-11-19T15:20:00
---
```

手动指定的日期会覆盖文件系统的时间戳。

## 🔍 搜索功能

### 快捷键

按 **Ctrl+K**（Mac: Cmd+K）呼出搜索框。

### 搜索范围

- 文章标题
- 文章标签
- 文章内容

### 使用方法

1. 按 `Ctrl+K` 打开搜索框
2. 输入关键词
3. 使用 `↑` `↓` 方向键选择结果
4. 按 `Enter` 打开文章
5. 按 `ESC` 关闭搜索框

### 搜索提示

- 实时搜索，即时显示结果
- 最多显示 10 条结果
- 关键词高亮显示
- 支持中英文搜索

## 🏷️ 标签系统

### 标签的作用

- 文章分类和组织
- Archive 页面按标签筛选
- 文章页面点击标签跳转到筛选页面

### 如何使用标签

**简单格式（推荐）**：
```yaml
tags: 技术, React, 教程
```

**数组格式**：
```yaml
tags:
  - "技术"
  - "React"
  - "教程"
```

### 标签交互

**在文章页面**：
- 点击标签可以跳转到 Archive 页面
- 自动筛选该标签的所有文章

**在 Archive 页面**：
- 顶部显示所有标签
- 点击标签筛选文章
- URL 会更新为 `#/archive?tag=标签名`
- 可以直接分享带标签的链接

### 标签命名建议

- 使用清晰、简洁的名称
- 保持一致性（避免同义词）
- 推荐标签：技术、文学、笔记、教程、项目等
- 可以使用中英文混合

## 📂 导航和页面结构

### 首页（Home）
- 显示最近 10 篇文章
- 提供 Archive 和 About 入口
- 支持键盘导航

### Archive 页面
- 显示所有普通文章
- 按标签筛选
- 网格布局展示
- 访问：`#/archive` 或点击首页 ARCHIVE 按钮

### About 页面
- 项目介绍
- 技术栈说明
- 联系方式
- 提供 CV 链接
- 访问：`#/about`

### CV 页面
- 独立的简历页面
- 不在 Archive 和首页显示
- 只能通过 About 页面访问
- 访问：`#/cv`

### 创建特殊页面（不在列表显示）

在 markdown frontmatter 中添加 `special: true`：

```yaml
---
title: CV
special: true
type: default
---
```

这样的文章：
- ✅ 可以通过 URL 直接访问（`#/post/cv`）
- ❌ 不会出现在首页文章列表
- ❌ 不会出现在 Archive 页面
- 适合：CV、隐藏页面、草稿等

## 📝 诗歌样式使用技巧

使用 `type: poetry` 时：

- **每一行诗**就是一个段落（在 Markdown 中用空行分隔）
- **诗节之间**多空一行（连续两个空行）会产生更大的间距
- 段落间距已经优化为很小（0.3rem），适合诗行
- 推荐使用中文楷体字体，自动应用

示例：

```markdown
---
type: poetry
---

# 诗歌标题

第一行诗

第二行诗

第三行诗

（这里空了两行，会产生诗节分隔效果）

第二节第一行

第二节第二行
```
