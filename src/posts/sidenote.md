
# 带有大量脚注的 Markdown 示例文档

> 这是一个演示脚注功能的 Markdown 文件，你可以直接保存为 `.md` 文件使用。[^0]

## 1. 引言：为什么要用脚注

在写作中使用脚注，可以把「想讲，但又不想打断主线」的内容移到页面底部。[^1]  
相比在正文中不断插入括号、破折号或补充说明，脚注让文本更「干净」[^2]，但又不牺牲信息量。[^3]

脚注还有几个典型用途：

1. 提供额外的背景说明（例如补充一个概念的历史来源）。[^4]
2. 标注引用来源，尤其是学术写作中。[^5]
3. 放一些「八卦」「旁白」或幽默感比较强的内容。[^6]

当然，脚注也可能被滥用：如果每一句话都有脚注，读者在正文和脚注之间来回跳转，反而会打断阅读节奏。[^7]

## 2. Markdown 脚注的基本语法

在大多数 Markdown 方言中，脚注语法大致如下：[^8]

- 在正文中插入脚注标记：`这是一个句子[^footnote-id]`
- 在文末定义脚注内容：`[^footnote-id]: 这里是脚注的内容`

例如：

> 有些人喜欢在行文中大量使用脚注[^fn-example]，有些人则更偏好简洁到底。

对应的脚注定义通常写在文末：[^9]

```text
[^fn-example]: 这是一个脚注的示例文本。
````

值得注意的是，不同的 Markdown 渲染器对脚注的支持并不完全一致。[^10]
在某些博客系统中，你需要启用特定插件或扩展才可以正常渲染脚注。[^11]

## 3. 脚注的排版与阅读体验

从排版角度看，脚注会对阅读体验产生几种影响：[^12]

1. **页面的视觉节奏**
   当脚注数量适中时，页面底部出现少量短行文字，可以形成一种「视觉锚点」。[^13]
   读者在阅读完一节后，下移视线查看脚注，反而会形成一种有节奏的阅读体验。[^14]

2. **信息的层次结构**
   正文承载「必须读」的信息，脚注承载「想读时可以读」的信息。[^15]
   这相当于在文中引入了一种轻量级的「可折叠内容」机制，只不过折叠方式依赖读者自己。[^16]

3. **跨屏阅读的负担**
   在纸质书中，脚注往往就在页面底部；但在手机或小屏幕设备上，多数阅读器会把脚注跳转到很远的位置，再跳回来。[^17]
   对某些读者而言，这是「信息丰富」；对另一些读者而言，则是「干扰源」。[^18]

因此，在为网页或电子书写作时，是否大量使用脚注，需要考虑目标载体与读者习惯。[^19]

## 4. 一些使用脚注的具体策略

### 4.1 把「元评论」放进脚注

所谓「元评论」，指的是对文本本身的评论，而不是对主题的评论。[^20]
例如，你在正文中讲一个理论，在脚注里可以说明「其实这个例子并不完美，但足够说明问题」。[^21]

这样做的好处是：

* 正文保持干净、线性；
* 脚注中可以适当放松语气，甚至调侃自己。[^22]

### 4.2 把技术细节和实现方案放进脚注

在写技术博客或教程时，常常需要说明某个实现细节、边界条件或兼容性问题。[^23]
这些内容对少数读者非常重要，对多数读者却是噪音。脚注刚好可以承担这种「分层」。[^24]

举例来说：

> 这个脚本在大多数现代浏览器中都可以直接运行[^compat]，但在老旧环境中可能需要一些 polyfill。[^25]

这时你可以在脚注里列出具体浏览器版本、兼容策略、相关链接等。[^26]

### 4.3 避免用脚注塞入「真正应该写在正文里」的内容

如果你发现脚注比正文还长，而且脚注里在展开一个关键论证，那说明这段内容应该回到正文里。[^27]
脚注更适合：

* 小范围补充；
* 对某个观点给出一两句额外解释；
* 延伸阅读的链接列表。[^28]

若一段脚注已经形成一个完整的小节，不妨直接升级为正文段落。[^29]

## 5. 与引用、参考文献的关系

脚注和参考文献并不互斥。[^30]
在学术写作中，一种常见做法是：正文中的脚注用于短评、解释；文末的参考文献列表用于系统性列出文献来源。[^31]

例如：

> 有学者指出，脚注本身也构成了一种「第二文本」，可以在边缘处与正文对话。[^32]

这里正文中的观点可以在脚注中进一步展开：

* 列出该学者的原文出处；
* 补充其理论背景；
* 简短评论其局限。[^33]

而在文末的参考文献部分，则以统一格式列出该文献的完整条目。[^34]

## 6. 一个带有多重脚注的示例段落

下面这个段落故意堆叠了多种脚注使用方式，便于你测试渲染效果：[^35]

> 在数字写作环境中，脚注的语用功能远比传统纸质时代更加丰富[^36]：它既是对正文的补充，也是对读者注意力的一种「温和干预」[^37]。当读者愿意主动点击脚注时，说明他们对当前话题的参与度已经提高[^38]；而当脚注中出现一些风格相对轻松的评论或旁白时，又可以在不破坏正文严肃性的前提下，为文本注入一点人味[^39]。更有甚者，有作者会在脚注中埋彩蛋[^40]，只有细读者才能看到，这本身也构成一种「读者之间的暗号」。[^41]

你可以留意以下几种用法：

* 同一个句子里可以有多个脚注标记；[^42]
* 脚注中可以包含链接、列表、甚至简单的 Markdown 格式；[^43]
* 只要脚注 ID 保持唯一，命名方式是自由的（例如 `[^a]`、`[^note-long]` 都可以）。[^44]

## 7. 实际写作中的几个小技巧

1. **先写正文，后写脚注**
   在起草初稿时，不必立刻决定哪些内容要放在脚注里。[^45]
   可以先用括号或高亮标记，等全文结构稳定后再统一整理成脚注。[^46]

2. **统一管理脚注编号**
   如果你写的是长文，建议定期检查脚注编号是否有重复或遗漏。[^47]
   某些编辑器或静态博客工具会在编译时自动处理编号，但手写时仍需保持谨慎。[^48]

3. **为脚注内容设定风格「边界」**
   例如，你可以约定：脚注只做如下几件事——解释术语、给出出处、偶尔调侃。[^49]
   其他所有内容一律回到正文中，以避免脚注变成「杂物间」。[^50]

## 8. 结语

脚注本质上是一种「把话说开一点」的写作工具。[^51]
它不会自动提升文章质量，但可以在尊重读者注意力的前提下，让文本多一些层次和密度。[^52]

只要记住一个简单原则：**脚注是为读者服务，而不是炫技**。[^53]
